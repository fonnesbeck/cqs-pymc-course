# `games_2025-04.parquet`

Game-level covariates corresponding to the same April 2025 window as `batted_balls_2025-04.parquet`. One row per (date, game) entry in the MLB schedule. Joinable to the batted-balls file on `game_pk` (preferred) or `game_date`.

- **Source:** MLB Stats API (`statsapi.mlb.com`) — `/api/v1/schedule` with `hydrate=gameInfo,venue,decisions,weather,team` plus a per-venue fetch of `/api/v1/venues/{id}?hydrate=location,timeZone,fieldInfo` for the 30 unique venues seen in the window.
- **Rows:** 400 (393 unique `game_pk`; see *Split-doubleheader duplicates* below)
- **Columns:** 53
- **Sort order:** `game_pk` ascending
- **Coverage:** 2025-04-01 → 2025-04-30, regular season only (`game_type = "R"`)

## Status of games in the file

| `status` | Count | Notes |
|---|---:|---|
| Final | 389 | Completed games — most useful for joining to batted-balls. |
| Postponed | 9 | No first pitch, no score, no weather populated. Will not join cleanly to batted-balls. |
| Completed Early | 2 | Official game ended before 9 innings (e.g. rain-shortened). Has a winner. |

## Fields

### Identity

| Column | Type | Description |
|---|---|---|
| `game_pk` | `Int64` | MLBAM game identifier. **Primary join key** to the batted-balls file. |
| `game_date` | `Date` | Official local-calendar date of the game (from the API's `officialDate`). Matches the Statcast `game_date` convention. |
| `season` | `Int64` | MLB season year (always 2025 here). |
| `game_type` | `String` | `"R"` = Regular season, `"S"` = Spring training, `"P"` = Postseason, etc. |

### Teams and outcome

| Column | Type | Description |
|---|---|---|
| `home_team_id` / `home_team_name` | `Int64` / `String` | Home team MLBAM ID and full name. |
| `away_team_id` / `away_team_name` | `Int64` / `String` | Away team MLBAM ID and full name. |
| `home_score` / `away_score` | `Int64` | Final score. Null for postponed games. |
| `home_is_winner` / `away_is_winner` | `Boolean` | Win flag. Null for postponed games. |
| `winning_pitcher` / `losing_pitcher` / `saving_pitcher` | `String` | Full name of the pitcher credited with W/L/SV. Null where not applicable (e.g. postponed games have all three null; tie games have no save). |

### Status and timing

| Column | Type | Description |

| `status_code` | `String` | One-letter code: `"F"` = final, `"P"` = postponed, `"O"` = other. |
| `first_pitch_utc` | `Datetime[μs, UTC]` | UTC timestamp of the first pitch (after any delay). Null for postponed games. |
| `game_duration_minutes` | `Int64` | Elapsed game time in minutes. Null for postponed games. |
| `delay_duration_minutes` | `Int64` | Total pre-game delay in minutes. |

| `scheduled_innings` | `Int64` | 9 for regulation, 7 for doubleheader games. |
| `double_header` | `String` | `"Y"` = single-admission DH, `"S"` = split DH, `"N"` = standalone. |
| `game_number` | `Int64` | 1 or 2 within a doubleheader. |
| `series_description` | `String` | e.g. `"Regular Season"`, `"Spring Training"`. |
| `series_game_number` | `Int64` | Position of this game within the home team's season series vs. this opponent. |
| `attendance` | `Int64` | Announced attendance. Null for postponed games. |

### Weather (gameInfo.weather block)

The MLB API returns one weather snapshot per game, captured near first pitch. Useful for outdoor parks; meaningless for retractable-roof stadiums (where the roof status determines whether the field is climate-controlled).

| Column | Type | Description |
|---|---|---|
| `weather_condition` | `String` | Free-form: `"Clear"`, `"Partly Cloudy"`, `"Cloudy"`, `"Overcast"`, `"Dome"`, etc. |
| `weather_temp_f` | `Int64` | Air temperature in degrees Fahrenheit (parsed from the API's string). Range observed: 34–97 °F. |
| `weather_wind_speed_mph` | `Int64` | Wind speed in mph (parsed from the wind string's leading integer). |
| `weather_wind_direction` | `String` | Direction phrase, e.g. `"In From CF"`, `"Out To LF"`, `"In From RF"`, `"L To R"`, `"R To L"`. |
| `weather_wind_raw` | `String` | Verbatim API string, e.g. `"6 mph, In From CF"`. Kept for traceability. |

### Venue — location

| Column | Type | Description |
|---|---|---|

| `venue_name` | `String` | Stadium name (e.g. `"Coors Field"`). |
| `venue_city` / `venue_state` / `venue_state_abbrev` | `String` | City, full state name, two-letter state code. |
| `venue_country` | `String` | ISO short name (`"USA"`, `"Canada"`). |
| `venue_postal_code` | `String` | Postal/ZIP code of the venue address. |
| `venue_latitude` / `venue_longitude` | `Float64` | WGS84 coordinates of the venue. |
| `venue_elevation_ft` | `Int64` | Elevation in feet above sea level (Coors Field = 5190). Coarsely a park-factor for carry distance. |
| `venue_azimuth_angle` | `Float64` | Compass bearing the pitcher faces, in degrees (0 = north). Useful for sun/weather exposure modeling. |
| `venue_capacity` | `Int64` | Stadium seating capacity. |
| `venue_time_zone` | `String` | **Not provided** by the MLB API even with hydration; left out of the schema. Use the `first_pitch_utc` plus a state→timezone lookup if you need it. |

### Venue — field geometry

All distances in feet, to the fence along the indicated ray. Use these to model park-factor effects on batted-ball outcomes (e.g. a 102 mph fly ball to dead center at Coors clears the 415 ft fence more often than at a 390 ft park).

| Column | Type | Description |
|---|---|---|
| `venue_left_line_ft` | `Int64` | Distance to the **left-field foul pole**. |
| `venue_left_ft` | `Int64` | Distance to left field (typically the LF wall, often deeper than the pole on symmetric parks). |
| `venue_left_center_ft` | `Int64` | Distance to left-center field. |
| `venue_center_ft` | `Int64` | Distance to straightaway center field. |
| `venue_right_center_ft` | `Int64` | Distance to right-center field. |
| `venue_right_ft` | `Int64` | Distance to right field. |
| `venue_right_line_ft` | `Int64` | Distance to the **right-field foul pole**. |

### Venue — surface and roof

| Column | Type | Description |
|---|---|---|
| `venue_turf_type` | `String` | `"Grass"` or `"Artificial Turf"`. |
| `venue_roof_type` | `String` | `"Open"` (outdoor), `"Retractable"` (closed for some games, open for others), or `"Fixed"` (dome). |

## Split-doubleheader duplicates

The MLB schedule endpoint returns 7 games twice each, for a total of 7 extra rows in this file (400 rows / 393 unique `game_pk`). These are games that were originally postponed and made up as the second game of a split doubleheader. The API emits two rows for the same `game_pk`:

1. The original entry, with `status = "Postponed"`, `game_number = 1`, `double_header = "N"`.
2. The makeup entry, with `status = "Final"`, `game_number = 2`, `double_header = "S"`.

For analysis use `status_code = "F"` to keep only completed games (this filters out the postponed rows automatically while keeping the makeup rows).

## Caveats and quirks

- **Weather is at first pitch only.** Conditions that change during the game (a day game where it warms up, a night game where the wind shifts) are not captured.
- **Retractable-roof games.** For 84 games played in retractable-roof stadiums (Tropicana Field, Rogers Centre, etc.), the weather block is recorded for "outside" conditions; the field conditions during the game depend on whether the roof was open. MLB doesn't expose the open/closed state on the schedule endpoint — you'd need the per-game `/feed/live` to recover it.
- **Time zone not in the API.** `venue_time_zone` is omitted from the schema; `timeZone` is null on `/api/v1/venues/{id}` even with `?hydrate=timeZone`. Use `first_pitch_utc` + a manual US/CA state→IANA-zone mapping if you need it downstream.
- **Indoor stadium temperatures.** Games at Tropicana Field, Rogers Centre, Chase Field, and Minute Maid Park still report a `weather_temp_f` (the outside temperature at the time), which is **not** the playing conditions. Combine with `venue_roof_type` before using as a covariate.
- **Dome status string.** A handful of older games return `weather_condition = "Dome"`. Treat as a missing/null signal.

## Suggested join

```python
import polars as pl

bb = pl.read_parquet("batted_balls_2025-04.parquet")
g  = pl.read_parquet("games_2025-04.parquet")

# Recommended: join on game_pk, keep only completed games.
joined = bb.join(
    g.filter(pl.col("status_code") == "F").select([
        "game_pk", "venue_name", "venue_elevation_ft",
        "venue_left_ft", "venue_center_ft", "venue_right_ft",
        "venue_roof_type", "venue_turf_type",
        "weather_temp_f", "weather_wind_speed_mph",
        "weather_wind_direction", "weather_condition",
        "first_pitch_utc", "day_night", "attendance",
    ]),
    on="game_pk",
    how="left",
)
```
